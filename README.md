# Лабораторная работа №6: Обратное проксирование через Nginx в Docker

## 📖 Описание
Данный проект демонстрирует настройку обратного прокси-сервера (reverse proxy) с использованием Nginx и Docker Compose. Все входящие HTTP-запросы на `80` порт хост-машины автоматически перенаправляются на веб-приложение, запущенное в изолированном Docker-контейнере.

## Технологический стек
- `Docker` & `Docker Compose` (v2+)
- `Nginx` (reverse proxy)


## Требования
- Установленные `Docker` и `Docker Compose`
- Свободный `80` порт на хост-машине

## Быстрый старт
## 1. Запустите окружение:
```
docker compose up -d
```

## 2. Проверьте работу прокси
```
curl -I http://localhost
```
Ожидаемый ответ: HTTP/1.1 200 OK

# Конфигурация
## proxy.conf

| Директива | Назначение |
|-----------|------------|
| upstream app_backend | Указывает на внутренний DNS-имя контейнера app |
| proxy_pass | Перенаправляет запросы из / в приложение |
| proxy_set_header | Передаёт реальный IP клиента, хост и протокол |

## docker-compose.yml
| Сервис/сеть | Назначение |
|-----------|------------|
| app | Целевое приложение (по умолчанию nginx:alpine для демонстрации) |
| proxy | Nginx, пробрасывающий 80:80 и монтирующий конфигурацию |
| lab_net | Изолированная bridge-сеть для безопасного взаимодействия контейнеров |

## Тестирование и диагностика
```
# Проверить статус контейнеров
docker compose ps

# Просмотреть логи прокси-сервера
docker compose logs proxy

# Проверить доступность приложения изнутри сети
docker compose exec proxy curl -s http://app
```
